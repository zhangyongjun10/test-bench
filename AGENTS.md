# Agent 编码规则

- 新增或修改代码时，每个类、方法、函数、接口、类型、常量等定义上方都必须添加中文注释。
- 注释需要说明该定义的职责、用途或关键约束，避免只重复名称本身。
- 如果定义涉及业务规则、执行链路、比对逻辑、Trace 处理、超时配置、并发隔离等关键行为，注释中必须写清楚核心规则和边界条件。
- 修改已有定义时，如果原定义缺少中文注释，需要在本次修改中补齐。
- 后台异步任务、延迟任务、补偿任务和重试任务不能只有 happy path 或裸 `return`；必须明确成功、失败、异常三类收口路径，并在失败或异常时更新相关业务状态、错误信息和完成时间，避免页面或数据库记录长期停留在处理中状态。
